What It Is For
- This file holds the current live state of the project.
- Write here where the project is, what it is trying to do, what is blocking it, and what the next sensible move is.
- It should help someone understand very quickly what is truly active now.

What Goes Here
- The current objective. It belongs here because it keeps work oriented instead of dispersing it. If you close it well, you gain direction.
- The dominant blocker of the moment. It belongs here because it names where the project is really stuck. If you name it well, you gain clearer priorities.
- A live view of the project in a few dense but readable lines. It belongs here because it helps show where the project is now. If you write it well, you gain continuity between sessions.
- Work already in progress. It belongs here because it shows what is already guiding the project in the present. If you keep it clean, you gain a live and readable file.
- Open items that are really weighing on the present. They belong here because they help show what is entering the work or what is blocking it. If you distinguish them well, you gain less noise.
- Questions that have real consequences. They belong here because they can truly change the project. If you close them well, you gain more useful openings.
- Live risks of the moment. They belong here because they show where the project can deform or stall. If you make them clear, you gain more lucidity.
- The next concrete and sensible move. It belongs here because it closes the present with a readable action. If you write it well, you gain operational continuity.

What Does Not Go Here
- The stable identity of the project. That belongs in 01 because it describes the project in a durable way, not its present. Otherwise you risk confusing orientation with live work. If you place it in 01, you gain a more readable present.
- Hard-to-reopen constants. That belongs in 02 because it belongs to the points you do not want to reopen easily. Otherwise you risk treating something stable as temporary. If you place it in 02, you gain a cleaner working file.
- Decisions already taken and already in effect. That belongs in 04 because it records what has already won. Otherwise you risk mixing present state and decision. If you place it in 04, you gain a clearer trace of what has already been decided.
- Strong proposals that have not yet truly entered the work. That belongs in 05 because it matters but is not yet part of the current live state. Otherwise you risk making a line look active when it is not. If you place it in 05, you gain a more honest 03.
- Repos, key folders, entrypoints, sensitive surfaces, minimal runbook, or critical dependencies, if you are describing them as technical map rather than live work. That belongs in 08 because it closes the real surface of the project. Otherwise you risk turning 03 into a technical map instead of a file about the present. If you place it in 08, you gain a more stable working file.
- A full history of everything that happened. That does not belong here because 03 is not an archive. Otherwise you risk turning it into a chronicle instead of a live view. If you keep it out, you gain clarity.
- An infinite backlog or a session diary. That does not belong here because it widens the file without helping read the present. Otherwise you risk losing the live node of the project. If you keep it out, you gain a more useful working file.

Update It When
- The operational picture truly changes.
- A line really enters current work.
- A blocker stops being secondary and becomes dominant.
- A session moves the project in a substantial way.
- Do not update it for every micro-move.

File Sections

Project Name
- project name:
- Write the name of the project this file serves.

Last Updated
- date:
- Use the date of the last substantial update.
- This is for reading freshness, not for building a full timeline.

Current Objective
- Write in one sentence what the project is trying to achieve now.
- It should orient work without going too deep into detail.

Current Blocker
- Write the node that is really slowing the project now.
- Not every problem: the one that weighs the most today.

Current State
- Write where the project is now in a few dense but readable lines.
- A line already in use may appear here even if it has not yet become a recorded decision.
- If you are listing already-closed decisions, you are probably writing in the wrong file.

Work In Progress
- Put only the lines of work that are actually active.
- If a line no longer has real energy, it should not stay here.
- If something stops being only an idea and starts guiding the work, it may appear here.

Open Items That Matter Now
- Name only what is about to enter the work or what is blocking the present.
- Do not turn this section into a second notes file.

Real Open Questions
- Write the questions or openings that have real impact.
- A real open question can change direction, grammar, product, or continuity.

Active Risks
- Only live risks belong here, not abstract fears.
- This section should show where the project can deform, slow down, or break if no action is taken.

Next Step
- Write the next concrete and sensible move.
- One clear move is better than an elegant but fake plan.

Session Close
- If a session changes the project, update this file before closing.
- If during the session a choice has already clearly won, update the decision file too.
- Before closing, realign at least: Current State, Work In Progress, Real Open Questions, Active Risks, Next Step.

Live Anchor Hygiene
- The section titles in 03 are operational anchors for reading, timeline, and maintenance; keep them stable.
- References to `D-xx` in `links to` must point only to real decisions.
- Open slots are acceptable while the file is still template-like; once 03 is truly live, they should not remain its normal form.
- If you need to cite useful files or material outside the canonical set, use `external reference` instead of widening `links to`.

Typical Errors
- Using 03 as an infinite diary.
- Treating it as a disguised decision log.
- Filling it with vague lines that do not help anyone understand the real moment.
- Putting strong ideas here that have not yet truly entered the work.
- Using it to pretend a line is live just because it feels like the right next move.
