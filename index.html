Signal Rail

A text-first governance system for human-AI project work.

Signal Rail separates project identity, live state, decisions, unresolved material, technical topology, continuity, findings, parking, and archive into different textual layers with different authority.

The result is not more notes. It is less collapse.

Open Repository Technical Overview

Problem

What Signal Rail is built to prevent

In long-running AI-assisted work, project material tends to flatten into one surface. Current work, decisions, unresolved ideas, technical notes, continuity scraps, and local findings start competing for the same authority.

  • identity turns into diary
  • decisions start sounding provisional
  • ideas get promoted too early
  • technical reality disappears into generic notes
  • continuity starts pretending to be truth

Model

Three system parts

Canonical layer

Authority-bearing files for project identity, freeze, live state, won decisions, unresolved gravity, and technical topology.

Support layer

Useful but intentionally non-canonical surfaces for continuity, field findings, parking, and archive.

Lateral kernel

A shared behavioral protocol for the agent that runs across the system without becoming host-project truth.

Kernel

`06_ai_to_ai.txt` is lateral by design

`06_ai_to_ai.txt` is not a canonical project file. It does not carry host-project truth. Its job is horizontal: activation, identity closure, minimum reads, modes, gates, validation, stop rules, and move order.

Each canonical can then define its own local `AI SECTION`. That local section governs the container itself. The kernel governs shared behavior across the Signal Rail surface.

06 = shared horizontal behavior

AI SECTION = container-local behavior

canonicals = project truth

support files = useful, non-canonical

Boundaries

The system relies on distinct surfaces

Baseline repository

The shipped source of the system itself.

Deployed instance

The tool surface used inside real work.

Host project

The actual governed project, which the instance must not infer automatically.

Bootstrap and localization

Preparation is separate from project truth

Bootstrap

`init_signal_rail.bat` and `init_signal_rail.ps1` prepare a fresh Signal Rail surface in a target folder. They do not define host-project truth.

Localization Kit

`SR Localization Kit` supports one structural pass: turning the fresh international baseline into one personal localized copy while keeping file names, IDs, marker name, system name, and core structure stable.

Signal Rail is the clean baseline system.
